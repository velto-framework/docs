Velto Roadmap

Velto is intentionally minimal in its initial versions. This roadmap outlines planned areas of evolution for the language, runtime, and tooling ecosystem.

Short‑term goals (v0.x)

  • Stabilise the core tag set.
  • Extend the behaviour attribute library.
  • Add configuration options for strict/relaxed parsing.
  • Improve diagnostics and error messages.

Mid‑term goals (v1.x)

  • Introduce a component model for reusable UI blocks.
  • Define a standard for scoped styles in Velto documents.
  • Support custom events and event bubbling.
  • Provide a CLI for building and bundling Velto applications.

Long‑term goals

  • Multiple runtime targets (for example server‑side, native, or hybrid).
  • Editor tooling and language server integration.
  • Formal specification with versioning and stability guarantees.
  • Pluggable extensions for third‑party tags and behaviours.

Versioning

Velto will use semantic versioning once it reaches v1.0.0. Until then, breaking changes may occur between minor versions as the language stabilises. Runtimes SHOULD document which Velto spec version they target.
