Runtime Contract

The runtime contract defines the minimum behaviour that any conforming Velto runtime MUST implement. Different runtimes may target different platforms (for example browsers, native, or server‑side rendering), but they must all respect the same language rules.

Loading

  • Runtimes MUST be able to load Velto documents from a text source.
  • Runtimes MUST support the .velto extension.
  • Runtimes MAY support additional sources (for example strings, streams, or in‑memory documents).

Parsing

  • Runtimes MUST parse Velto input as XML or XML‑compatible syntax.
  • Runtimes MUST reject malformed documents and report an error.
  • Runtimes MUST locate and use the <velto> root element.

Tag handling

  • Runtimes MUST implement the standard tag mappings defined in the Tags section.
  • Runtimes MAY support additional, custom tags.
  • Runtimes MUST provide a sensible fallback for unknown tags (for example mapping to <div>).

Attribute handling

  • Runtimes MUST support behaviour attributes defined in the Attributes section.
  • Runtimes MUST pass unknown attributes through to the platform layer unchanged.
  • Runtimes MAY emit warnings for unknown or suspicious attributes in development mode.

Event handling

  • Runtimes MUST attach event handlers corresponding to behaviour attributes.
  • Runtimes MUST ensure that events fire in response to user interactions as specified.
  • Runtimes MAY provide configuration to enable or disable specific behaviours.

Error reporting

  • Runtimes MUST log fatal errors when rendering cannot proceed.
  • Runtimes SHOULD log non‑fatal warnings for unknown tags and attributes.
  • Runtimes MAY expose debug hooks for developer tools.
