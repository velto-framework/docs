Velto Attributes

Attributes in Velto fall into three broad categories: structural attributes, presentation attributes, and behaviour attributes. This section defines how attributes are named, how they are interpreted, and how runtimes should handle unknown attributes.

Naming rules

  • Attribute names MUST be lowercase.
  • Attribute names MAY contain hyphens (-) and colons (:).
  • Behaviour attributes SHOULD use the form on-event:action.

Structural attributes

Structural attributes describe identity and structure, and generally map directly to HTML attributes.

  • id – unique identifier within the document.
  • class – space‑separated list of classes.
  • data-* – custom data attributes passed through to the DOM.
<text1 id="intro" class="lead">
  Welcome to Velto
</text1>

Presentation attributes

Presentation attributes control visual appearance. Velto does not define its own styling model; instead, it passes presentation attributes to the underlying platform.

  • style – inline CSS styles.
  • Any other attributes understood by the target HTML element.
<text2 style="color: #888; font-size: 0.9rem;">
  Secondary text
</text2>

Behaviour attributes

Behaviour attributes trigger interactive behaviour, usually in response to events. They follow the form on-event:action and are interpreted by the Velto runtime.

on-click:redirect

Redirects the browser to the given URL when the element is clicked. 

<button on-click:redirect="https://example.com">
  Go to example
</button>

on-click:alert

Shows a native alert dialog with the provided message. 

<button on-click:alert="Button clicked!">
  Click me
</button>

on-click:log

Logs a message using the Velto runtime's logging facilities. 

<button on-click:log="User clicked the button">
  Log action
</button>

Unknown attributes

Unknown attributes (those not recognised as behaviours) MUST be passed through to the underlying DOM element. Runtimes MAY log a warning in development mode, but MUST NOT remove or modify unknown attributes.
